Iraquin

A High Performance Layer-1 for Decentralized Compute & Verifiable AI Governance

What is Iraquin?

Iraquin is a Layer-1 blockchain designed for high throughput decentralized compute and AI inference, without relying on on chain smart contract execution or total transaction ordering.

The core idea is simple:

Instead of re executing everything on chain, Iraquin commits cryptographically verified state transitions produced off chain.

This enables horizontal scalability, eliminates duplicated work, and avoids reorgs while preserving correctness through verification, economic accountability, and dispute resolution.

Core Architecture

Iraquin is built around two coupled protocols:

Prime â€” High Performance Block Production

  • Separates work between miners and colliders
  • Miners handle cryptographic verification
  • Colliders handle state execution
  • Eliminates duplicated computation
  • Supports parallel block production over disjoint account sets
  • Blocks are optimistically accepted for immediate throughput

Omega â€” Verification, Disputes & Finality

  • Validator quorum verifies Prime blocks asynchronously
  • Detects invalid state updates or behavior
  • Applies forward correction (no rollbacks, no reorgs)
  • Provides epoch bounded dispute windows
  • Finalizes state once disputes close

Execution Model (No VM)

  • No on-chain smart contract VM
  • Contracts execute off-chain inside TEEs
  • Written in any language
  • Execution produces:
    • State diff
    • Read/write account set
    • Cryptographic attestation
  • The chain verifies and commits the state update

This model avoids VM bottlenecks and is natively compatible with AI workloads.

Native Compute & AI Marketplace

  • Built in marketplace for compute and AI inference
  • Jobs are submitted via a consensus backed Data Availability (DA) layer
  • Workers bid, stake collateral, and commit to delivery deadlines
  • Results are verified and settled on chain
  • Failed or invalid jobs are slashed and re auctioned

Verifiable AI Governance (OPML)

  • AI models propose consensus-level decisions, including:
    • Fees
    • Throughput tuning
    • Burn and stake parameters
    • Network configuration
  • Decisions are:
    • Cryptographically verifiable (OPML)
    • Challengeable
    • Enforced by consensus
  • AI acts as a provable decision proposer, not a trusted authority

Why This Matters

  • No reorgs or rollbacks
  • No duplicated execution
  • Parallel throughput without sharding
  • AI native by design
  • Suitable for DePIN, compute networks, and inference heavy protocols

Iraquin is not a "better VM chain."

It's a different execution and governance model for AI native decentralized infrastructure.
